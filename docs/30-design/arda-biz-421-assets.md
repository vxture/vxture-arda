# 数据资产 功能设计（arda-biz-421-assets）

> 状态：功能层 · 功能设计（待评审）
> 模板/看板：[`biz-400`](arda-biz-400-functions.md)；数据模型：[`data-210`](arda-data-210-assets.md)
> 与 [`biz-422 元数据`](arda-biz-422-metadata.md) 是**两个维度**：资产 = **对象/单元**（有哪些数据、归谁、多少）；元数据 = **描述**（数据长什么样、什么意思）。

---

## 1. 功能定义

把数据登记、组织为**可管理的资产单元（对象）**：资产身份、归属、主题域/团队、资产目录与发现。关注**"有哪些数据、归谁、多少"**。（描述性的 schema/术语/标签属**元数据** `biz-422`。）

## 2. 贯通链（目标 → 过程 → 结果 → 服务 → 监管）

| 环 | 做什么 | 靠什么实现 | 接下一环 |
|---|---|---|---|
| **目标·定义** | 编目体系 / 资产分类 / 主题域·团队 facet | `Dataset` 身份 + `domain`/`team`/`classification` | 资产模型 → 供登记 |
| **过程·执行** | 登记（from 接入）、组织、归属指派、资产级分级 | `Dataset` CRUD、`ownerUserId`、`classification`(AssetLevel) | 资产 → 供发现/画像 |
| **结果·看** | 资产目录（列表/卡片/域筛选）、**资产画像**（聚合各功能结果面）、有多少/容量画像 | 目录查询 + 画像聚合 + `sizeBytes` 汇总 | 画像 → 供服务选资产、供消费发现 |
| **服务·用** | 目录供发现、画像供服务发布选资产 | 目录对外（`biz-441`） | — |
| **监管·审计** | 资产创建/归属/分级变更审计 | `AuditLog{action: asset.change}` | 审计流水 |

## 3. 断链清单

| 编号 | 断链（环） | 现状 | 接通方案 | 依赖 |
|---|---|---|---|---|
| `A-BL1` | 结果：画像"结果面聚合"未装配 | 画像要聚合质量/血缘/安全/主数据/标准（含元数据），各功能结果环未接 | 画像聚合装配（随各功能结果环接通逐步点亮） | `biz-422`/`biz-431`/`biz-432`/`biz-433`/`biz-434`/`biz-435` |
| `A-BL2` | 结果：有多少/容量画像缺 | 存储占用未聚合展示 | 聚合 `sizeBytes` + 配额 | 存储 |
| `A-BL3` | 目标：资产分类体系薄 | domain/team 为自由键，无受控分类树 | 评估受控分类（可选） | — |

> 元数据相关断链（术语/标签管理、语义检索、列级 Field）移至 [`biz-422`](arda-biz-422-metadata.md)。

## 4. 数据模型（da delta）

- **已建**：`Dataset`（资产身份 + facet + `classification`）。
- **派生（不落库）**：质量分、订阅数、容量汇总。
- **实现要点**：画像 = 聚合视图（读多功能实体，含元数据）；`code` ws 内唯一。

## 5. 依赖

| 依赖 | 用途 | 断链 |
|---|---|---|
| 接入（`biz-410`） | 生成 Dataset | 上游 |

| 质量/血缘/安全/标准主数据 | 画像结果面 | A-BL1 |
| 服务（`biz-441`） | 目录对外发现 | — |

## 6. 门控（能力键）

- 浏览目录：`arda.assets.catalog`（**baseline**，域可见性基准）。
- 编辑资产（归属/分级）：`arda.assets.edit_metadata`；批量：`arda.assets.bulk_ops`（写操作，`editor+`）。
- 配额：`arda.quota.datasets`、`arda.quota.storage_bytes`。
