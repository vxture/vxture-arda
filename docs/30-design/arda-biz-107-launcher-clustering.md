# Launcher 归集与域聚合 (arda-biz-107-launcher-clustering)

> 状态: 看和组织层 · 两层归集设计 (owner 已定稿, 2026-07-18)
> 范围: 在 [`biz-106`](arda-biz-106-domain-menus.md) 的 15 域 L2 菜单骨架之上, 做两件事:
> (1) launcher 从纵向长列改为横向分组网格; (2) 把 15 个域聚合为 13 个菜单, 解散
> "数据治理"独立菜单。不改变门控能力键定义。
> 上游: [`biz-105`](arda-biz-105-capability-map.md), [`biz-106`](arda-biz-106-domain-menus.md),
> [`biz-110`](arda-biz-110-domains.md) (角色动线)

---

## 0. 两层机制

- **第 1 层 - launcher 面板 (LAUNCHER_GROUPS)**: 15 个域纵向平铺太长, 改为**横向分组
  网格** (mega-panel), 5 个组横排, 每组下挂若干板块 tile。参照 DataWorks/IDMC/Purview
  的"分类式产品切换器"形态, 不是纵向长列。
- **第 2 层 - 域聚合 (BOARDS)**: 把关系近、同角色反复切换的域**合并成同一个菜单**
  (共用一套侧边栏)。owner 钉死 8 个域独立, 规划+架构合并, 其余我方分析。

## 1. owner 决策 (2026-07-18, 承 biz-106)

| # | 决策 | 结论 |
|---|---|---|
| 1 | 资产做落脚点 | 删除独立"总览"板块; **数据资产**成为全员默认落地页, `数据总览`并入并改名`资产总览`。所有板块名称去掉"域"字 |
| 2 | 集成/建库/开发 | 在"接入与开发"组内拆为**三个独立菜单**: `数据集成`(链接外部数据) / `数据建库`(自有数据入库) / `数据开发`(数据生成数据); 产物统一是数据产品/数据资产 |
| 3 | 数据治理 | **彻底解散, 不留降级壳**。平台本身即围绕治理展开, 治理的活儿要么是设计成果 (进`规划设计`), 要么是运行监管 (进`数据运营`), 要么已分散在各专业域 (标准/质量/安全/主数据)。不存在"只属于治理、又不是设计、又不在专业域"的第三类日常工作 |
| 4 | 运营/管理的"管理" | = **系统管理** (平台/系统级: 审批中心/密钥/审计/参数/通知 + 深链 vxture 平台), 非数据管理。改名`管理`->`系统管理`, 与数据学科的`主数据管理`/`元数据管理`区分 |

## 2. 治理"工作内容"拆解 (决策 3 的依据)

| 治理活动 | 性质 | 归处 |
|---|---|---|
| 治理组织、责任人、管理体系、制度流程 | 设计成果 (建一次少改) | -> 规划设计`治理体系`组 |
| 成熟度评估、考核体系 | 设计/评估框架 | -> 规划设计`评估考核`组 |
| 问题管理、整改跟踪、治理评价 | 运行工作 (持续/跨域) | -> 数据运营`治理运营`组 |
| 标准评审、质量整改、安全审批、主数据 steward | 运行工作 | 已在各专业域工作台 |

治理概念不丢: launcher `治理与管控`组名保留、规划设计有`治理体系`、数据运营有`治理运营`
- 三处露出治理, 只是不再有空壳独立入口。

## 3. 第 1 层: 横向 launcher (5 组 / 13 板块)

```
 资产与服务      规划与设计     接入与开发     治理与管控       运营与管理
 ─────────      ─────────     ─────────     ──────────       ─────────
 数据资产 ★      规划设计       数据集成       数据质量         数据运营
 数据服务        数据标准       数据建库       数据安全         系统管理
                              数据开发       主数据管理
                                            元数据管理
      ★ = 全员默认落脚点 (资产总览)
```

`LAUNCHER_GROUPS` = `assetService / planDesign / ingestDev / governControl / operateAdmin`。
`Board.group` 指派归组; header 渲染为 `.vxh-launcher-grid` 多列网格 (console.css),
tile 紧凑化 (仅名称, 描述移入 title 提示)。

## 4. 第 2 层: 15 域 -> 13 板块

| Launcher 组 | 板块 | 来自原 15 域 | 依据 |
|---|---|---|---|
| 资产与服务 | 数据资产 (含资产总览/工作待办) | 总览 + 数据资产 | 总览并入 (决策1) |
| 资产与服务 | 数据服务 | 数据服务 | 独立 |
| 规划与设计 | 规划设计 (含治理体系) | 数据规划 + 数据架构 + 治理框架 | 合并 + 治理设计并入 |
| 规划与设计 | 数据标准 | 数据标准 | 独立 |
| 接入与开发 | 数据集成 | 数据集成 | 独立 (决策2) |
| 接入与开发 | 数据建库 | 新增 | 自有入库 (决策2) |
| 接入与开发 | 数据开发 | 数据开发 | 独立 (决策2) |
| 治理与管控 | 数据质量 | 数据质量 | 独立 |
| 治理与管控 | 数据安全 | 数据安全 | 独立 |
| 治理与管控 | 主数据管理 | 主数据 | 独立 + "管理"二字 |
| 治理与管控 | 元数据管理 | 元数据 | 独立 + "管理"二字 |
| 运营与管理 | 数据运营 (含治理运营) | 数据运营 + 治理运行 | 治理运营并入 |
| 运营与管理 | 系统管理 | 管理 | 独立, 改名 (决策4) |

原 15 域中 `总览`并入数据资产、`数据规划`+`数据架构`合并为规划设计、`数据治理`解散,
新增 `数据建库`; 净 15 -> 13。
